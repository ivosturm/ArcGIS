Combiner les couches

Cet outil combine deux couches d’entités pour créer une seule couche en sortie. Les couches à combiner doivent avoir le même type de géométrie et le même type temporel. Vous pouvez déterminer la manière dont les champs des couches en entrée sont rassemblés et copiés. Par exemple :

  • Vous disposez de trois couches pour l'Angleterre, le pays de Galles et l'Ecosse et vous souhaitez une seule couche pour la Grande-Bretagne.
  • Vous disposez de 2 couches et chacune contient des informations sur les parcelles des cantons contigus. Vous souhaitez les rassembler en une seule couche en ne conservant que les champs qui portent le même nom et qui ont le même type sur les 2 couches en entrée.

Choisir une couche

Couche contenant des entités tabulaires, ponctuelles, linéaires ou surfaciques avec laquelle la deuxième couche d’entités sera fusionnée. Seuls les champs de cette couche seront inclus dans la couche de résultat.

Choisir la couche à fusionner

Couche d’entités à fusionner avec la couche en entrée. La couche à fusionner doit posséder le même type de géométrie et le même type temporel que la couche en entrée.

Modifier les champs en fusion

Par défaut, tous les champs des deux couches en entrée seront inclus dans la sortie. Vous pouvez modifier la manière dont les champs issus de la couche à fusionner seront inclus dans la sortie par le biais des options suivantes :

  • Renommer : le champ est renommé dans la sortie.
  • Supprimer : le champ est supprimé de la sortie.
  • Apparier : les noms de champs sont rendus équivalents. Les valeurs du champ de la couche à combiner sont alors copiées dans le champ en sortie apparié.

Si les deux couches contiennent un champ de nom et de type de données identiques, leur contenu est fusionné en un même champ de ce nom. Cela ne s’applique pas aux champs où Apparier est utilisé dans le paramètre Modifier les champs en fusion.

Nom de la couche de résultat

Nom de la couche à créer. Si vous écrivez sur une instance ArcGIS Data Store, les résultats seront enregistrés dans My Content (Mon contenu) et ajoutés à la carte. Si vous écrivez sur un partage de fichiers Big Data, les résultats seront enregistrés dans le partage de fichiers Big Data et ajoutés à son manifeste. Il ne sera pas ajouté à la carte. Le nom par défaut repose sur le nom de l'outil et sur le nom de la couche en entrée. Si la couche existe déjà, l'outil échoue.

Lors de l’écriture sur ArcGIS Data Store (répertoire de données relationnelles ou spatio-temporelles) via la zone de liste déroulante Save result in (Enregistrer le résultat dans), vous pouvez spécifier le nom d’un dossier dans My Content (Mon contenu) où le résultat sera enregistré.
