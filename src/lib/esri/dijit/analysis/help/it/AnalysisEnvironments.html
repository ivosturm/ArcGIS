Impostazione dell'ambiente di analisi nel Portal

Impostazioni d'ambiente per l'analisi in Portal for ArcGIS e ArcGIS Online.

Sovrascrivi layer risultati

Specifica se sovrascrivere i layer esistenti. Quando l'opzione è selezionata, se nei contenuti personali esiste già un layer con lo stesso nome, questo verrà sovrascritto.

  • Selezionata: se nei contenuti personali esiste già un layer con lo stesso nome, questo verrà sovrascritto. L'output avrà lo stesso sistema di coordinate dell'input.
  • Deselezionata: se esiste già il nome del layer dei risultati specificato, si verificherà un errore e il layer dei risultati dovrà essere rinominato prima che lo strumento venga eseguito. Questa è l'impostazione predefinita.

Chiudi widget di analisi dopo l'esecuzione dell'analisi

Specifica se mantenere la finestra di dialogo dell'analisi aperta e abilitata mentre lo strumento è in esecuzione. Questa impostazione consente di apportare modifiche ai parametri di analisi mentre continua l'esecuzione del lavoro di analisi precedente. L'analisi è un processo iterativo e, mantenendo aperta la finestra di dialogo, consente di modificare i parametri ed eseguire di nuovo l'analisi.

  • Selezionata: la finestra di dialogo dello strumento di analisi verrà chiusa e verrà di nuovo visualizzato il sommario. Si tratta dell'impostazione predefinita.
  • Deselezionata: la finestra di dialogo dello strumento di analisi rimarrà aperta e abilitata.

Memorizza i risultati delle analisi come feature layer ospitati

Specifica se il layer dei risultati creato dall'analisi sarà una raccolta di feature o un feature layer hosted.

  • Selezionata: i risultati dell'analisi creeranno un feature layer hosted e verrà creato un elemento nei contenuti personali. Questa è l'impostazione predefinita.
  • Deselezionata: i risultati dell'analisi verranno creati come raccolta di feature.

Sistema di coordinate di output

Specifica il sistema di coordinate del layer dei risultati per l'analisi raster.

  • Lo stesso come input: il risultato dell'analisi avrà lo stesso sistema di coordinate dell'input. Questa è l'impostazione predefinita.
  • Come specificato: il risultato dell'analisi avrà il sistema di coordinate prescelto. Quando questa opzione è selezionata, fare clic sul pulsante del globo e scegliere da un elenco di sistemi di coordinate noti o aggiungere il WKID del riferimento spaziale nella casella fornita.
  • Layer <nome>: il risultato dell'analisi avrà lo stesso sistema di coordinate di un layer esistente scelto nella Web Map.

Elaborazione del sistema di coordinate

Specifica il sistema di coordinate che verrà utilizzato dall'analisi durante l'esecuzione dell'analisi di GeoAnalytics.

Per GeoAnalytics, alcune analisi richiedono che l'elaborazione venga eseguita in un sistema di coordinate proiettato. Tutti gli strumenti che hanno implementato il binning dei dati o delle unità lineari richiedono un sistema di coordinate proiettato.

  • Lo stesso come input: l'analisi verrà elaborata nello stesso sistema di coordinate dell'input. Questa è l'impostazione predefinita.
  • Come specificato: l'analisi avrà il sistema di coordinate prescelto. Quando questa opzione è selezionata, fare clic sul pulsante del globo e scegliere da un elenco di sistemi di coordinate proiettati noti o aggiungere il WKID del riferimento spaziale nella casella fornita.
  • Layer <nome>: l'analisi avrà lo stesso sistema di coordinate di un layer esistente scelto nella Web Map.

Estensione

Specifica l'estensione o il confine che verrà utilizzato durante l'esecuzione dell'analisi. Tutte le feature di input che sono completamente all'interno o che intersecano l'estensione specificata verranno utilizzate nell'analisi.

  • Predefinito: l'estensione fornita dallo strumento.
  • Come specificato: l'estensione è definita dalle coordinate fornite.
  • Layer <nome>: l'estensione usata per elaborare l'analisi sarà la stessa dell'estensione spaziale di un layer esistente scelto nella Web Map.

Snap raster

Regola l'estensione del layer raster di output in modo che corrisponda all'allineamento delle celle del layer Snap raster specificato nell'analisi raster.

Dimensione cella

Specifica la dimensione della cella o la risoluzione che verrà utilizzata per creare il layer raster di output nell'analisi raster. La risoluzione di output predefinita è determinata dalla dimensione di cella più grande del layer raster di input.

  • Numero minimo di input: utilizzare la dimensione cella più piccola di tutti i layer di input.
  • Numero massimo di input: utilizzare la dimensione cella più grande di tutti i layer di input. Questa è l'impostazione predefinita.
  • Come specificato: specificare un valore numerico per impostare la dimensione della cella. Se questa opzione è selezionata, il valore predefinito è 1.
  • Layer <nome>: imposta la dimensione della cella sul layer raster scelto.

Maschera

Specifica un layer che sarà usato per definire l'area di interesse nell'analisi raster. Solo le celle che rientrano nella maschera di analisi verranno considerate nell'operazione di analisi.

  • La maschera può essere un raster o un feature layer.
  • Se la maschera di analisi è un raster, tutte le celle contenenti un valore verranno considerate per definire la maschera. Le celle comprese in un raster usato come maschera che hanno un valore NoData verranno considerate esterne alla maschera e avranno valore NoData anche nel layer dei risultati dell'analisi.
  • Se la maschera dell'analisi è un feature layer, sarà convertito internamente in un raster al momento dell'esecuzione. Per questo motivo, assicurarsi che Dimensione cella e Snap raster siano impostati appropriatamente per l'analisi.

Archivio dati

I risultati di GeoAnalytics possono essere memorizzati o in ArcGIS Data Store ed esposti come un feature layer in Portal for ArcGIS o in una condivisione file Big Data.

Quando si memorizza un risultato in ArcGIS Data Store, nella maggior parte dei casi i risultati dovrebbero essere memorizzati nello Spatiotemporal Data Store. Si tratta dell'impostazione predefinita. Di seguito sono riportate le ragioni per cui sarebbe preferibile memorizzare i risultati in un Data Store relazionale:

  • Per utilizzare i risultati nella collaborazione tra portali
  • Per abilitare funzionalità di sincronizzazione con i risultati

Non utilizzare un Data Store relazionale se si prevede un incremento dei risultati GeoAnalytics e si desidera sfruttare le funzionalità dello Spatiotemporal Big Data Store per gestire grandi quantità di dati.

Fattore di elaborazione parallela

Distribuisce l'analisi su varie istanze del servizio Elaborazione raster.

Gli strumenti che rispettano l'ambiente Fattore di elaborazione parallela consentono di controllare il numero di istanze del servizio di elaborazione raster utilizzabili per l'elaborazione dei dati.

Se lo strumento non rispetta il Tipo di processore o se l'ambiente Tipo di processore è impostato su CPU, l'ambiente Fattore di elaborazione parallela controlla le istanze del servizio di elaborazione raster (CPU). Se il Tipo di processore è impostato su GPU, l'ambiente Fattore di elaborazione parallela controlla il numero di istanze di elaborazione raster sulla GPU.

Impostando il Fattore di elaborazione parallela, è possibile richiedere il numero di lavoratori che il server dell'immagine dell'analisi raster usa per elaborare una singola attività di analisi raster. Tuttavia, se il numero totale di processi paralleli supera il numero massimo di istanze del servizio di elaborazione raster (CPU o GPU), i processi paralleli aggiuntivi saranno messi in coda.

Per impostazione predefinita, il Fattore di elaborazione parallela non è specificato; in questo caso, lo strumento userà l'80% del numero massimo di istanze del servizio di elaborazione raster. È possibile specificare tanto un numero intero quanto una percentuale come fattore di elaborazione parallela.

Tipo di processore

Gli strumenti che rispettano l'ambiente Tipo di processore consentono di scegliere dove e come elaborare i dati. È possibile elaborare i dati usando la CPU o la GPU del computer del server. Se l'ambiente di Tipo di processore è vuoto, lo strumento usa la CPU per elaborare i dati. Questa è l'impostazione predefinita.

L'elaborazione sulla CPU può essere svolta in parallelo su vari core e istanze, sottostando alla gestione del Fattore di elaborazione parallela.

Le GPU sono efficaci per l'elaborazione di grafiche e immagini, in quanto la loro struttura fortemente parallela le rende efficienti per l'elaborazione di grandi blocchi di dati in maniera ripetitiva. Lo strumento di analisi raster che rispetti questo ambiente può distribuire il proprio lavoro su istanze della GPU su vari computer con server di analisi raster, sottostando alla gestione del Fattore di elaborazione parallela.

  • GPU: i dati saranno elaborati utilizzando l'unità di elaborazione grafica.
  • CPU: i dati saranno elaborati utilizzando l'unità di elaborazione centrale del computer.
