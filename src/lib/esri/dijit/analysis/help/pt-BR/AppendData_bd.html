Acrescentar Dados

Esta ferramenta acrescenta dados a uma camada de feição hospedada existente. Anexar Dados modifica a camada de entrada original e não gera uma nova camada de saída. Você pode combinar campos com base no nome de campo e tipo de campo, ou você pode aplicar métodos de combinação mais avançados.

Os seguintes são exemplos:

  • Você é funcionário de uma grande seguradora que gera registros de vendas mensalmente. Para o seu relatório anual, os dados mensais devem ser compilados para análise. Para iniciar a análise de vendas, você pode aproveitar a ferramenta Anexar Dados para atualizar sua camada AnnualReport_2018 assim que seus números mensais forem disponibilizados. Já que você sabe que sua organização mantém um esquema consistente a cada mês, você pode utilizar o método de mapeamento de campo padrão para adicionar feições de forma eficiente com base no nome e no tipo de campo.
  • Dez grupos ambientais, cada um, coleta centenas de milhares de medições climáticas dentro de sua jurisdição. Estas organizações desejam compilar seus dados para entender melhor os padrões regionais e globais. Cada organização determina o esquema para seu conjunto de dados. Cada um deles especificou diferentes nomes de campo para dados de temperatura e velocidade do vento. Dadas as diferenças do nome de campo entre conjuntos de dados, o mapeamento de campo pode ser aproveitado para especificar como adicionar medições de temperatura e velocidade do vento aos seus respectivos campos.

Escolher como anexar campos (opcional)

Especifica como os campos da camada de anexo são acrescentados na camada de entrada. Por padrão, todos os campos de anexo e entrada correspondentes serão anexados. Os campos são uma combinação se tiverem o mesmo nome de campo e o mesmo tipo de campo. Opcionalmente, você pode escolher como os campos são combinados ou calcular as expressões Arcade para preencher os valores dos campos.

Por exemplo, você tem uma camada de entrada como os campos field_number, field_string, date_field, unused_field e uma camada de anexo com os campos field_string, field_number e MY_DATE_FIELD .

  • Por padrão, a ferramenta combinará field_number na camada de entrada para field_number na camada de anexo, e field_string na camada de entrada para field_string na camada de anexo.
  • Por padrão, os campos date_field e unused_field da camada de entrada serão preenchidos com valores de null pois eles não têm nenhum campo correspondente na camada de anexo.
  • Opcionalmente, você pode acrescentar valores aos campos que não foram correspondidos automaticamente por padrão. Por exemplo, date_field na camada de entrada pode ser correspondido ao campo de anexo MY_DATE_FIELD. Para fazer isto, utilize o menu suspenso ao lado de date_field na primeira coluna e selecione MY_DATE_FIELD no menu suspenso da segunda coluna.
Como outro exemplo, uma camada de entrada tem campos numéricos value_1, value_2, value_3 e my_Average, e uma camada de anexo tem campos numéricos value_1, value_2 e value_3.
  • Por padrão, value_1, value_2 e value_3 serão correspondidos e anexados.
  • Além do comportamento padrão, os valores podem ser calculados para o campo my_Average . Faça isto utilizando vários campos e operações matemáticas selecionando a opção suspensa Expressão na coluna ao lado de my_Average e construa uma expressão Arcade personalizada, como 
    average($feature["value_1"] + $feature["value_2"] + $feature["value_3"])
    .
