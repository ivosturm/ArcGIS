Anexar Dados

Esta ferramenta anexa dados a uma camada de elementos alojada existente. Anexar dados modifica a camada de entrada original e não gera uma nova camada de saída. Pode fazer corresponder campos com base no nome de campo e tipo de campo, ou pode aplicar métodos de correspondência mais avançados.

Os seguintes são exemplos:

  • É funcionário de uma grande companhia de seguros que gera registos de vendas mensais. Para o seu relatório anual, os dados mensais têm de ser compilados para análise. Para iniciar a análise de vendas, pode tirar partido de Anexar Dados para atualizar a sua camada AnnualReport_2018 após os seus números mensais estarem disponíveis. Como sabe que a sua organização mantém um esquema mensal consistente. Pode utilizar o método de field mapping predefinido para anexar elementos com base no nome e tipo de campo.
  • Dez grupos ambientalistas recolheram, cada um, centenas de milhares de medições climatéricas nas suas jurisdições. Estas organizações pretendem compilar os seus dados para melhor compreender padrões regionais e globais. Cada organização determina o esquema do seu conjunto de dados. Cada um deles especificou diferentes nomes de campos para dados de temperatura e velocidade do vento. Dadas as diferenças de nomes de campo entre conjuntos de dados, cartografia de campos pode ser aproveitada para especificar como anexar medições de temperatura e velocidade do vento aos seus respetivos campos.

Anexar todos os elementos ou definir um subconjunto (opcional)

Identifica os elementos a anexar. Este parâmetro é opcional.

Selecionar como anexar campos (opcional)

Especifica como os campos da camada anexar são anexados à camada de entrada. Por predefinição, todos os campos de entrada e anexar correspondentes serão anexados. Os campos correspondem caso tenham o mesmo nome de campo e o mesmo tipo de campo. Opcionalmente, pode escolher como os campos são correspondidos, ou calcular expressões Arcade para popular valores de campos.

Por exemplo, tem uma camada de entrada com os campos field_number, field_string, date_field, unused_field , e uma camada anexar com os campos field_string, field_number, e MY_DATE_FIELD .

  • Por predefinição, a ferramenta fará corresponder field_number na camada de entrada a field_number na camada anexar, e field_string na camada de entrada a field_string na camada anexar.
  • Por predefinição, os campos camada de entrada date_field e unused_field serão populados com valores de null porque não têm campo correspondente na camada anexar.
  • Opcionalmente, pode anexar valores a campos que não foram correspondidos automaticamente por predefinição. Por exemplo, date_field na camada de entrada pode ser correspondido com o campo anexar MY_DATE_FIELD. Para fazer isto, utilize o menu pendente ao lado de date_field na primeira coluna e selecione MY_DATE_FIELD no menu pendente na segunda coluna.
Para dar outro exemplo, uma camada de entrada tem campos numéricos value_1, value_2, value_3, e my_Average, e uma camada anexar tem campos numéricos value_1, value_2, e value_3.
  • Por predefnição, value_1, value_2, e value_3 serão correspondidos e anexados.
  • Para além do comportamento padrão, podem ser calculados valores para o campo my_Average . Faça isto utilizando múltiplos campos e operações matemáticas, selecionando a opção pendente Expressão na coluna ao lado de my_Average e construindo uma expressão Arcade personalizada, tal como 
    average($feature["value_1"] + $feature["value_2"] + $feature["value_3"])
    .
