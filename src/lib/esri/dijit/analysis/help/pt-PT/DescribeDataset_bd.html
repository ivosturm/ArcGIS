Descrever Conjunto de Dados

A ferramenta Descrever Conjunto de Dados proporciona uma perspetiva mais detalhada do seu conjunto de dados. Antes de executar análises sobre grandes quantidades de dados, utilize esta ferramenta para determinar quais são e onde estão os seus dados de entrada. Por defeito, o resultado será uma string JSON que indica os descritores principais da sua camada de entrada e uma tabela que especifica as estatísticas de resumo para cada campo.

Opcionalmente, pode criar camadas de elementos de saída adicionais para descrever detalhadamente os seus dados. As opções são uma camada de amostra ou uma camada de extensão. Pode optar por produzir uma ou ambas.

Por exemplo, suponhamos que recebe um grande volume de ficheiros de big data que contém 15 conjuntos de dados. Cada conjunto de dados tem 10 milhões de elementos de área que representam edifícios e casas em diferentes regiões do seu país. A sua tarefa consiste em analisar a região C, mas não sabe qual o conjunto de dados que contém estes elementos. Para descobrir qual o conjunto de dados que deverá utilizar, execute Descrever Conjunto de Dados e escolha criar uma camada de extensão para investigar qual o conjunto de dados que está na sua região de estudo.

Um outro exemplo... imagine que tem como tarefa realizar um fluxo de trabalho de análise sobre um grande volume de dados. Quer testar o fluxo de trabalho, mas pode demorar horas ou dias com o seu conjunto de dados completo. Em vez de utilizar tempo e recursos a executar a análise completa, crie uma camada de amostra para testar eficientemente o seu fluxo de trabalho completo.

Escolher conjunto de dados a descrever

A camada que contém elementos de pontos, linhas, áreas ou tabulares que serão descritos, resumidos ou incluídos na amostra.

Para além de escolher uma camada do seu mapa, pode escolher Escolher Camada de Análise na parte inferior da lista pendente para navegar até aos seus conteúdos de um conjunto de dados de partilha de ficheiros de big data ou camada de elementos.

Compreender o seu conjunto de dados criando uma (opcional)

Produzir camadas descritivas adicionais para melhorar a sua compreensão dos seus big data. Pode escolher produzir zero, uma ou ambas as camadas seguintes:

  • Camada de extensão—cria um elemento de área que representa a extensão dos seus elementos de entrada ou área de interesse.
  • Camada de amostra—cria uma camada de subconjunto que contém uma quantidade específica de elementos de entrada a partir do seu conjunto de dados ou área de interesse.

Camada de amostra

Produzir uma camada que contém um subconjunto de elementos da sua camada de entrada. Se Camada de amostra estiver selecionada poderá especificar o número de elementos que pretende devolver na camada de amostra. Por defeito, serão devolvidos 100 elementos de amostra na camada de saída.

O valor tem de ser maior que zero. Se especificar um número superior ao número total de elementos todos os elementos serão devolvidos.

Se Utilizar extensão de mapa atual estiver selecionada, então a camada de amostra irá conter elementos do interior da extensão atual do mapa.

Nome da camada resultante

O nome da camada que será criada. Se estiver a escrever para ArcGIS Data Store, os seus resultados serão gravados em O Meu Conteúdo e adicionados ao mapa. Se estiver a escrever para uma partilha de ficheiros big data, os seus resultados serão guardados numa partilha de ficheiros big data e adicionados ao seu manifesto. Não serão adicionados ao mapa. O nome padrão é baseado no nome da ferramenta e do nome da camada de entrada. Caso a camada já existe, a ferramenta irá falhar.
