Beräkna densitet

Med verktyget Beräkna densitet kan du skapa en densitetskarta av punktgeoobjekt genom att sprida ut kända antal av något fenomen (som representeras som attribut för punkterna) över kartan. Resultatet blir ett lager med områden klassificerade från lägsta till högsta densitet.

För punktdata bör varje punkt representera platsen för en händelse eller incident, medan resultatlagret representerar antalet incidenter per enhetsområde. Ett högre densitetsvärde på en ny plats innebär att det finns fler punkter nära den platsen. I många fall kan resultatlagret tolkas som ett riskområde för framtida händelser. Om indatapunkterna till exempel representerar platser för blixtnedslag, kan resultatlagret tolkas som ett riskområde för framtida blixtnedslag.

Andra exempel på användningsområden för det här verktyget är bland annat:

  • Skapa kartor över brottslighet som hjälper polisen att fördela sina resurser till områden med hög brottslighet.
  • Beräkna densitet för sjukhus inom en kommun. Resultatlagret visar områden med hög och låg tillgång till sjukhus, och den här informationen kan användas för att avgöra var nya sjukhus bör byggas.
  • Identifiera områden där risken för skogsbränder är stor baserat på platsen för tidigare skogsbränder.
  • Hitta orter som ligger långt från större motorvägar för att kunna planera var nya vägar bör byggas.

Välj ett punktlager som densiteten ska beräknas från

Det punktlager på vilket densiteten ska beräknas.

Förutom att välja ett lager från kartan kan du välja Välj analyslager längst ned i listrutan för att bläddra till ditt innehåll efter ett big data-fildelningsdataset eller geoobjektlager.

Välj ett eller flera fält att beräkna densitet från (valfritt)

Ange ett eller flera fält som visar antalet incidenter på respektive plats. Om du till exempel har punkter som representerar städer, kan du använda ett fält som representerar befolkningen i staden som ett antalsfält. Då räknar det resulterande befolkningstäthetslagret med större befolkningstäthet nära städer med större befolkning.

Densiteten för antalet punkter beräknas alltid. Om inga fält väljs, anses varje plats representera ett enda antal.

Välj en lagerplatsform

Formen på klasserna som punkterna aggregeras till och som används för beräkning av densiteten.

Analyser som använder lagringsplatserna Kvadrat eller Hexagon kräver ett projicerat koordinatsystem. Du kan ange Bearbetande koordinatsystem i Analysmiljö. Om ditt bearbetande koordinatsystem inte är inställt som ett projicerat koordinatsystem, blir du uppmanad att ställa in det när du använder Kör analys .

Välj lagerplatsstorlek för aggregering

Storleken på de genererade klasserna. För en kvadratisk lagringsplats motsvarar storleken kvadratens höjd. För en hexagon motsvarar storleken hexagonens höjd (från parallella sidor).

Välj densitetsviktning att tillämpa

Vilken typ av viktning som tillämpas på densitetsberäkningen. Det finns två alternativ:

  • Enhetlig – Beräknar magnituden per område.
  • Kärna – Tillämpar en kärnfunktion för att passa in en jämnt avsmalnande yta till varje punkt.

Välj närområdets storlek

Storleken på området som används för att beräkna densiteten. Närområdets storlek måste vara större än lagerplatsstorleken.

Välj utdataenheter för densitet

Önskade utdataenheter för densitetsvärdena. Som standard är detta inställt på kvadratkilometer eller kvadratmiles beroende på vilka standardenheter som anges i din profil. Om densitetsvärdena är mycket små kan du öka storleken för områdesenheterna (till exempel kvadratmeter till kvadratkilometer) för att returnera större värden. Värdena skalanpassas bara till resultatet.

Beräkna densitet med tidssteg (valfritt)

Om tid är aktiverat på indatapunktlagret och det är av typen tidspunkt, kan du analysera med tidsstegning. Du kan ställa in tre parametrar när du använder tid:

  • Tidsstegsintervall
  • Hur ofta tidssteget ska upprepas
  • Tid som används till att justera tidssteg till

Om du till exempel har data som representerar ett års tid och du vill analysera dem i veckovisa steg, ställer du in Tidsstegsintervall på 1 vecka.

Om du till exempel har data som representerar ett års tid och du vill analysera dem med hjälp av den första veckan i månaden, ställer du in Tidsstegsintervall på 1 vecka, Hur ofta tidssteget ska upprepas på 1 månad och Tid som används till att justera tidssteg till på 1 januari, kl. 00:00.

Tidsstegsintervall

Det tidsintervall som används för att generera tidssteg. Tidsstegsintervall kan användas var för sig eller med parametern Hur ofta tidssteget ska upprepas eller Tid som används till att justera tidssteg till.

Om du till exempel vill skapa tidssteg som inträffar varje måndag från 09:00 till 10:00, ställer du in Tidsstegsintervall på 1 timme, Hur ofta tidssteget ska upprepas på 1 vecka och Tid som används till att justera tidssteg till på 9:00:00 på en måndag.

Hur ofta tidssteget ska upprepas

Det steg som används för att beräkna ett tidssteg. Hur ofta tidssteget ska upprepas kan användas var för sig, med Tidsstegsintervall, med Referenstid eller med både Tidsstegsintervall och Tid som används till att justera tidssteg till.

Om du till exempel vill skapa tidssteg som inträffar varje måndag från 09:00 till 10:00, ställer du in Tidsstegsintervall på 1 timme, Hur ofta tidssteget ska upprepas på 1 vecka och Tid som används till att justera tidssteg till på 9:00:00 på en måndag.

Tid som används till att justera tidssteg till

Datum och tid som används för att justera tidsdelar. Tidssteg startar vid och fortsätter bakåt från den här tiden. Om ingen referenstid väljs justeras tidssteg mot 1 januari 1970.

Om du till exempel vill skapa tidssteg som inträffar varje måndag från 09:00 till 10:00, ställer du in Tidsstegsintervall på 1 timme, Hur ofta tidssteget ska upprepas på 1 vecka och Tid som används till att justera tidssteg till på 9:00:00 på en måndag.

SpatialReference (wkid)

Det här är en tillfällig parameter för förhandsversionen för att ställa in den geografiska referensen för bearbetning. Många big data-verktyg kräver att ett projicerat koordinatsystem används som geografisk referens för bearbetning. Som standard använder verktyget indatakoordinatsystemet men misslyckas om det är ett geografiskt koordinatsystem. Ange WKID för att ställa in ett projicerat koordinatsystem. Till exempel anges Web Mercator som 3857.

Välj datalager

GeoAnalytics-resultat sparas i ett datalager och visas som ett geoobjektlager i Portal for ArcGIS. I de flesta fall ska resultat sparas i det rumstemporala datalagret och det är standardinställningen. I vissa fall är det ett bra alternativ att spara resultat i det relationella datalagret. Följande är anledningar till varför du kanske vill spara resultat i det relationella datalagret:

  • Du kan använda resultaten i portal-till-portal-samarbeten.
  • Du kan aktivera synkroniseringsfunktioner med dina resultat.

Du bör inte använda det relationella datalagret om du förväntar dig att dina GeoAnalytics-resultat kommer att öka och du behöver använda det rumstemporala big data-lagrets funktioner till att hantera stora mängder data.

Resultatlagrets namn

Namnet på lagret som ska skapas. Om du skriver till en ArcGIS Data Store, sparas dina resultat i Mitt innehåll och läggs till i kartan. Om du skriver till en big data-fildelning, lagras dina resultat i big data-fildelningen och läggs till i dess manifest. Det läggs inte till i kartan. Standardnamnet baseras på verktygets namn och indatalagrets namn. Om lagret redan finns misslyckas verktyget.

När du skriver till ArcGIS Data Store (relationellt eller rumstemporärt Big Data-datalager) med listrutan Spara resultat i kan du ange namnet på en mapp i Mitt innehåll där resultatet ska sparas.
