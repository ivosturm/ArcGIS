Copier dans le stockage des données

Cet outil copie une couche en entrée dans un stockage des données. Vous pouvez copier les résultats sur ArcGIS Data Store (répertoire de données Big Data spatio-temporelles ou relationnelles), et une couche sera générée dans votre portail ou vous pouvez copier les résultats sur un partage de fichiers Big Data.

Vous pouvez par exemple copier des entités stockées dans un partage de fichiers Big Data dans un stockage des données relationnelles et spécifier que seules les entités au sein de l'étendue actuelle de la carte seront copiées. Ceci crée un service d'entités hébergé avec uniquement les entités qui se trouvaient au sein de l'étendue de la carte spécifiée.

Choisir la couche à copier

La couche ou table qui va être copiée.

Outre le fait de choisir une couche de votre carte, vous pouvez sélectionner Choose Analysis Layer (Choisir une couche d’analyse) au bas de la liste déroulante pour parcourir votre contenu et rechercher une couche d’entités ou un jeu de données de partage de fichiers Big Data.

Vous pouvez utiliser toutes les entités ou effectuer une sélection (facultatif)

Vous avez également la possibilité d'utiliser le bouton de sélection pour identifier les emplacements à copier.

Choisissez le répertoire de données de destination de la copie

Le stockage des données dans lequel la couche va être copiée. Ce paramètre remplacera la sélection dans les environnements d’analyse (Analysis Environments). Les options sont les suivantes :

  • ArcGIS Data Store ArcGIS Data Store
    • Spatio-temporels : il s'agit de la valeur par défaut.
    • Relationnels
  • Modèle en sortie du partage de fichiers de Big Data
    • Liste des modèles de partage de fichiers Big Data disponibles au format bigDataFileShareName-templateName. Les modèles de partages de fichiers Big Data doivent être configurés avant l’exécution de l’outil.

Nom de la couche de résultat

Nom de la couche à créer. Si vous écrivez sur une instance ArcGIS Data Store, les résultats seront enregistrés dans My Content (Mon contenu) et ajoutés à la carte. Si vous écrivez sur un partage de fichiers Big Data, les résultats seront enregistrés dans le partage de fichiers Big Data et ajoutés à son manifeste. Il ne sera pas ajouté à la carte. Le nom par défaut repose sur le nom de l'outil et sur le nom de la couche en entrée. Si la couche existe déjà, l'outil échoue.

Lors de l’écriture sur ArcGIS Data Store (répertoire de données relationnelles ou spatio-temporelles) via la zone de liste déroulante Save result in (Enregistrer le résultat dans), vous pouvez spécifier le nom d’un dossier dans My Content (Mon contenu) où le résultat sera enregistré.
