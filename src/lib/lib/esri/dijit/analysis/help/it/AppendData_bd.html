Aggiungi dati

Questo strumento aggiunge dati ad un feature layer hosted esistente. Aggiungi dati modifica il layer di input originale e non genera un nuovo layer di output. È possibile abbinare i campi in base al nome del campo e al tipo del campo o è possibile applicare metodi di corrispondenza più avanzati.

Di seguito sono riportati degli esempi:

  • L’utente è un dipendente di una grande compagnia di assicurazioni che genera mensilmente record di vendita. Per il report annuale, i dati mensili devono essere compilati per l'analisi. Per avviare un'analisi delle vendite, è possibile utilizzare Aggiungi dati per aggiornare il layer AnnualReport_2018 una volta disponibili i numeri mensili. Poiché si sa che l’organizzazione mantiene uno schema coerente ogni mese, è possibile utilizzare il metodo di mappatura dei campi predefinito per aggiungere in modo efficiente le feature in base al nome e al tipo del campo.
  • Dieci gruppi ambientali raccolgono centinaia di migliaia di misurazioni climatiche nell'ambito della loro giurisdizione. Queste organizzazioni desiderano compilare i loro dati per comprendere meglio i modelli regionali e globali. Ogni organizzazione determina lo schema per il proprio dataset. Ciascuno ha specificato nomi di campo diversi per i dati relativi alla temperatura e alla velocità del vento. Date le differenze nel nome del campo tra i dataset, è possibile sfruttare la mappatura dei campi per specificare come aggiungere le misurazioni di temperatura e velocità del vento nei rispettivi campi.

Identifica le feature da aggiungere. Questo parametro è opzionale.

Scegli come aggiungere campi (opzionale)

Specifica il modo in cui i campi del layer aggiunti vengono aggiunti al layer di input. Per impostazione predefinita, verranno aggiunti tutti i campi di input e di aggiunta corrispondenti. I campi sono una corrispondenza se hanno lo stesso nome del campo e lo stesso tipo del campo. Facoltativamente, è possibile scegliere la modalità di corrispondenza dei campi o calcolare le espressioni Arcade per compilare i valori dei campi.

Ad esempio, si dispone di un layer input con campi field_number, field_string, date_field, unused_field e un layer di aggiunta con campi field_string, field_number e MY_DATE_FIELD.

  • Per impostazione predefinita, lo strumento fa corrispondere field_number nel layer di input a field_number nel layer di aggiunta e field_string nel layer di input a field_string nel layer di aggiunta.
  • Per impostazione predefinita, il layer di input date_field e i campi unused_field vengono compilati con i valori di null in quanto non dispongono di campi corrispondenti nel layer di aggiunta.
  • Facoltativamente, è possibile aggiungere valori ai campi non automaticamente corrispondenti per impostazione predefinita. Ad esempio, date_field nel layer di input può essere abbinato al campo di aggiunta MY_DATE_FIELD. A tal fine, utilizzare il menu a discesa accanto a date_field nella prima colonna e selezionare MY_DATE_FIELD dal menu a discesa nella seconda colonna.
In un altro esempio, un layer di input dispone di campi numerici value_1, value_2, value_3 e my_Average e un layer di aggiunta dispone di campi numerici value_1, value_2 e value_3.
  • Per impostazione predefinita, value_1, value_2 e value_3 vengono abbinati e aggiunti.
  • Oltre al comportamento predefinito, i valori possono essere calcolati per il campo my_Average. Effettuare questa operazione utilizzando vari campi e operazioni matematiche selezionando l’opzione a discesa Espressione nella colonna accanto a my_Average e creando un’espressione Arcade personalizzata, quale 
    average($feature["value_1"] + $feature["value_2"] + $feature["value_3"])
    .
