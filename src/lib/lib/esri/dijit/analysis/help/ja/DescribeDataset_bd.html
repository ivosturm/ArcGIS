データセットの記述 (Describe Dataset)

[データセットの記述 (Describe Dataset)] ツールを使用すると、データセットをより詳細に考察できます。大量のデータに対する解析を実行する前に、このツールを使用して、入力データの内容と場所を確認します。デフォルトでは、入力レイヤーの主要記述子の概略を示す JSON 文字列と、フィールドごとのサマリー統計の詳細を示すテーブルが結果として出力されます。

必要に応じて、データについてさらに詳しく記述する追加の出力フィーチャ レイヤーを作成できます。このオプションには、サンプル レイヤーと範囲レイヤーがあり、 いずれか 1 つまたは両方を出力できます。

たとえば、15 のデータセットが格納されているビッグ データ ファイル共有があるとします。それぞれのデータセットには、国内各地にある建物と住居を表す 1,000 万のエリア フィーチャが含まれています。割り当てられたタスクは地域 C の解析ですが、これらのフィーチャがどのデータセットに含まれているか不明です。どのデータセットを使用すべきか判断するには、[データセットの記述 (Describe Dataset)] を実行し、範囲レイヤーの作成を選択して、どのデータセットが分析範囲内にあるかを調べます。

もう 1 つの例として、大量のデータに対する解析ワークフローの完了というタスクが割り当てられたとします。ワークフローを実行するにあたり、データセット全体を処理するには数時間または数日かかる可能性があります。この場合、解析全体の実行に時間とリソースを使用する代わりに、サンプル レイヤーを作成し、それを使用して全体的なワークフローを効率的にテストします。

記述するデータセットの選択

記述、集計、サンプリングの対象となるポイント、ライン、エリア、または表形式フィーチャを含むレイヤー。

マップからレイヤーを選択できるだけでなく、ドロップダウン リストの下部にある [解析レイヤーの選択] を選択し、ビッグ データ ファイル共有データセットまたはフィーチャ レイヤーのコンテンツを参照して選択することもできます。

追加の記述レイヤーの作成によるデータセットの理解 (オプション)

ビッグ データに対する理解を深めるために、追加の記述レイヤーを出力します。次に示すレイヤーのうち、いずれか 1 つまたは両方を出力できます。あるいは、どちらも出力しないことを選択できます。

  • 範囲レイヤー - 入力フィーチャ、つまり対象エリアの範囲を表すエリア フィーチャを作成します。
  • サンプル レイヤー - データセット、つまり対象エリア内から、指定された数の入力フィーチャを含むサブセット レイヤーを作成します。

サンプル レイヤー

入力レイヤーからフィーチャのサブセットを含むレイヤーを出力します。 [サンプル レイヤー] が選択された場合は、サンプル レイヤー内で返されるフィーチャの数を指定できます。デフォルトでは、100 個のサンプル フィーチャが出力レイヤー内で返されます。

0 より大きい値を指定する必要があります。フィーチャの合計数より大きい数を指定すると、すべてのフィーチャが返されます。

[現在のマップ範囲を使用] が選択されている場合、サンプル レイヤーにはマップの範囲内にあるフィーチャが含まれます。

レイヤーの範囲

入力フィーチャの範囲を表す 1 つのフィーチャを含むフィーチャ レイヤーを出力するには、 [範囲レイヤー] ボタンを選択します。これはデフォルトで選択されています。

[現在のマップ範囲を使用] が選択されている場合、範囲レイヤーはマップの表示範囲を表します。

結果レイヤー名

作成されるレイヤーの名前。 ArcGIS Data Store に書き込んでいる場合、結果は [マイ コンテンツ] に保存され、マップに追加されます。ビッグ データ ファイル共有に書き込んでいる場合、結果はビッグ データ ファイル共有に格納され、そのマニフェストに追加されます。この場合、結果はマップに追加されません。 デフォルトの名前は、ツール名と入力レイヤー名に基づいて設定されます。レイヤーがすでに存在する場合、このツールは正常に実行されません。
