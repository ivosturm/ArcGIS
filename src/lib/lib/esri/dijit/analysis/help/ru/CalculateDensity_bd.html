Вычислить плотность

Инструмент Подсчитать плотность создает карту плотности из точечных пространственных объектов посредством разброса по карте известного количества событий (представленных как атрибуты точек). Результатом является слой из полигонов, классифицированных от наименьшей плотности к наибольшей

Каждая входная точка должна представлять местоположение некоторого события или инцидента, а полученный слой результатов будет представлять некоторое множество событий на единицу площади. Более высокое значение плотности в новом местоположении означает, что вблизи этого местоположения находится большее количество точек. Зачастую этот полученный слой может рассматриваться как зона (поверхность) повышенной вероятности для будущих событий. Например, если входные точки представляют местоположения ударов молний, то полученный слой может рассматриваться как поверхность вероятности ударов молний в будущем.

Этот инструмент используется также в следующих случаях:

  • Создание карт, отображающих концентрацию правонарушений, что помогает полиции грамотно распределять ресурсы в областях с высоким уровнем преступности.
  • Расчет плотности больниц в административном районе. Слой результатов покажет районы с высокой и низкой доступностью больниц, и эта информация может быть использована при принятии решения о том, где необходимо строить новые больницы.
  • Определение районов с высоким уровнем риска возникновения лесных пожаров на основе исторических данных о местах их возникновения.
  • Поиск удаленных от основных дорог населенных пунктов для планирования мест, где должны быть построены новые дороги.

Выбрать точечный слой для вычисления плотности

Точечный слой, в котором будет вычисляться плотность.

Кроме выбора слоя из вашей карты, можно щелкнуть Выбрать слой анализа в нижней части ниспадающего списка, чтобы найти свои ресурсы для набора данных или векторного слоя файлового хранилища больших данных.

Выбрать одно или несколько полей для вычисления плотности (дополнительно)

Укажите одно или несколько полей, определяющее число инцидентов в каждом местоположении. Например, если у вас города представлены точками, то в качестве поля количества вы можете использовать поле, представляющее население города; полученный слой плотности населения покажет более высокие плотности населения вблизи городов с большей численностью населения.

Плотность плотность количества точек будет вычисляться всегда. Если поля не выбраны, предполагается, что каждое местоположение представляет одну единицу количества.

Выбрать форму бина

Форма бинов, в которые собираются точки, используемые для вычисления плотности.

Анализ, использующий квадратные или шестиугольные бины, требует системы координат проекции. Вы можете настроить Систему координат обработки в Параметрах среды анализа. Если система координат обработки не является системой координат проекции, при Запуске анализа вам предложат изменить её. .

Выбор размера бина для агрегации

Размер создаваемых бинов. Для квадратного бина, размер соответствует высоте квадрата. Для шестиугольного, размер соответствует высоте шестиугольника (по параллельным сторонам).

Выбрать взвешивание плотности для применения

Тип взвешивания, применяемый в вычислении плотности. Существует две опции:

  • Универсальный - вычисляет количество на единицу площади.
  • Ядро - использует функцию ядра для подгонки плавно сужающейся поверхности к каждой точке.

Выбрать размер окрестности

Размер области, используемой для вычисления плотности. Размер окрестности должен быть больше размера бина.

Выбрать выходные единицы измерения для плотности

Желаемые единицы измерения выходных значений плотности. По умолчанию используются кв. километры или кв. мили, в зависимости от единиц измерения, заданных в вашем профиле. Если значения плотности очень малы, можно увеличить размер единиц площади (например, кв. метры на кв. километры), чтобы получить большие значения. Эти значения приводят только к масштабированию результата.

Вычислить плотность с помощью временных шагов (дополнительно)

Если на входном точечном слое используется текущее время, можно выполнять анализ с использованием временных шагов. Имеется три параметра, которые можно задать при использовании времени:

  • Интервал временного шага
  • Как часто повторять временной шаг
  • Время для выравнивания временных шагов

Например, если у вас есть данные. Которые представляют собой значения за год и вы хотите проанализировать их с недельным шагом, укажите Интервал временного шага как 1 week.

Например, если у вас есть набор данных, который представляет s значения за год и вы хотите проанализировать их по каждой первой неделе месяца, укажите Интервал временного шага как 1 неделя, Как часто повторять временной шаг как 1 месяц и Время для выравнивания временных шагов как January 1, at 12:00 am.

Интервал временного шага

Интервал времени, который используется для построения временных шагов. Интервал временного шага может использоваться сам по себе или с параметрами Как часто повторять временной шаг или Время для выравнивания временных шагов.

Например, если вы хотите создать временные шаги, которые происходят каждый понедельник с 9:00 до 10:00 утра, задайте Интервал временного шага как 1 час, Как часто повторять временной шаг как 1 неделя и Время для выравнивания временных шагов как 9:00:00 a.m. on a Monday.

Как часто повторять временной шаг

Шаг, использующийся для вычисления временного шага. Как часто повторять временной шаг может быть использован сам по себе, с Интервал временного шага, с Базовым временем или с обоими Интервал временного шага и Время для выравнивания временных шагов.

Например, если вы хотите создать временные шаги, которые происходят каждый понедельник с 9:00 до 10:00 утра, задайте Интервал временного шага как 1 час, Как часто повторять временной шаг как 1 неделя и Время для выравнивания временных шагов как 9:00:00 в понедельник.

Время для выравнивания временных шагов до

Дата и время, используемое для выравнивания временных срезов. Временные шаги начнутся св и будут отсчитываться назад от этого времени. Если базовое время выбрано, временные шаги будут выровнены по 1 января, 1970.

Например, если вы хотите создать временные шаги, которые происходят каждый понедельник с 9:00 до 10:00 утра, задайте Интервал временного шага как 1 час, Как часто повторять временной шаг как 1 неделя и Время для выравнивания временных шагов как 9:00:00 в понедельник.

Пространственная привязка (wkid)

Это временный параметр предварительной версии для задания обработки пространственной привязки. Для многих инструментов больших данных необходимо, чтобы пространственная привязка для обработки была определена в системе координат проекции. По умолчанию инструмент будет использовать входную систему координат, но если это географическая система координат – произойдет сбой. Чтобы задать систему координат проекции, введите WKID. Например, для Web Mercator следует задать 3857.

Выбрать хранилище данных

Результаты GeoAnalytics хранятся в хранилище данных и выглядят как векторный слой в Portal for ArcGIS. В большинстве случаев результаты следует сохранять в хранилище пространственно-временных данных, это настройка по умолчанию. В некоторых случаях , сохранение результатов в реляционном хранилище данных является хорошим решением. Следующее может быть поводом для сохранения результатов в реляционном хранилище данных:

  • Вы можете использовать результаты на портале-для-совместной работы.
  • Вы можете включить синхронизацию для результатов.

Вы не должны использовать реляционные хранилища данных, если ожидаете , что результаты GeoAnalytics увеличатся, и хотите использовать преимущества возможностей хранилища больших пространственно-временных данных для управления большими объемами данных.

Имя слоя результата

Имя создаваемого слоя. Если вы записываете в ArcGIS Data Store, ваши результаты будут сохранены в Моих ресурсах и добавлены на карту. Если вы записываете в файловое хранилище больших данных, ваши результаты будут сохранятся в файловом хранилище больших данных и добавляться в его файл манифеста. Они не будут добавлены на карту. Имя слоя по умолчанию зависит от имени инструмента и имени входного слоя. Если слой уже существует, произойдет сбой.

При записи в ArcGIS Data Store (реляционное или пространственно-временное хранилище больших данных) с помощью ниспадающего списка поля Сохранить результат в вы можете задать имя папки в разделе Мои ресурсы, в которую будет записан результат.
