Заново создать треки

Этот инструмент работает со слоем с поддержкой времени, содержащим точечные или полигональные объекты, которые представляют текущий момент времени. Он сначала определяет, какие объекты принадлежат треку, используя идентификатор. Используя время в каждом местоположении, треки последовательно упорядочиваются и трансформируются в линии или полигоны, отображающие путь перемещения с течением времени. Дополнительно входные данные могут быть буферизованы по полю, что позволит создать в каждом местоположении n полигон. Эти буферизованные точки или входные полигоны, которые затем последовательно соединяются, создавая трек в виде полигона, ширина которого зависит от атрибута интереса. Полученные треки имеют время начала и окончания, соответствующие первому и последнему по времени объектам в данном треке. При создании треков вычисляется статистика по входным объектам, которая присваивается выходному треку. Чаще всего определяется число точек в пределах полигона, но также может вычисляться и другая статистика.

Объекты слоев с поддержкой времени могут быть представлены двумя способами:

  • Текущий момент – один момент времени
  • Интервал – время начала и время окончания

Например, предположим у вас есть GPS-координаты урагана, получаемые каждые 10 минут. Каждое GPS-измерение записывает название урагана, его положение, время записи и скорость ветра. С этой информацией можно создать треки каждого урагана, используя имя для его идентификации. Кроме того, вы можете вычислить такую статистику как средняя, минимальная и максимальная скорость ветра каждого урагана, а также число измерения по каждому треку.

Используя тот же пример, можно создать буфер треков по скорости ветра. Инструмент записывает в буфер поле скорости ветра в каждом местоположении, соединяет местоположения из буфера вместе и создает полигон пути урагана с учетом изменений скорости ветра.

Выбрать объекты для создания треков из

Точечный или полигональный слой, который будет перестроен в треки. Входным слоем должен быть слой с включенным временем, с объектами, представляющими определенный момент времени. У слоя должна быть система координат проекции, или при помощи Параметров среды анализа пространственная привязка обработки должна быть задана как система координат проекции, если применяется буфер.

Кроме выбора слоя из вашей карты, можно щелкнуть Выбрать слой анализа в нижней части ниспадающего списка, чтобы найти свои ресурсы для набора данных или векторного слоя файлового хранилища больших данных.

Выберите поля для идентификации треков

Поля, представляющие идентификатор трека.

Например, если вы создаете треки ураганов, в качестве поля трека можно использовать его имя.

Выбрать метод для создания треков

Метод, используемый для объединения треков и наложения буфера (если применимо). Метод Плоский быстрее вычисляет результаты, но не может построить треки, пересекающие линию перемены дат или учитывающие при буферизации реальную форму земли. Метод Геодезический может создавать треки, пересекающие линию перемены дат, и использует геодезический буфер с учетом формы земли.

Создать выражение для построения буфера входных объектов (дополнительно)

Выражение используются для вычисления буферного расстояния вокруг входных объектов. Это выражение может быть сгенерировано с помощью калькулятора буфера, поддерживаются основные операции, такие как сложение, вычитание, умножение и деление. Значения вычисляются с использованием системы координат анализа. У слоя должна быть система координат проекции, или при помощи Параметров среды анализа пространственная привязка обработки должна быть задана как система координат проекции, если применяется буфер.

Выражение вида $feature.windspeed * 1000 применит буфер размером 1 000, умноженная на поле windspeed. В калькуляторе буфера можно использовать несколько полей.

Также можно задать отслеживаемое выражение. Например, можно вычислить сумму значений поля windspeed для текущего объекта и двух предыдущих объектов с выражением типа $track.field(windspeed).history(-3). В каждой локации сумма текущих windspeed и двух предыдущих измерений будет вычислена и буферизована.

Выбрать время для разбиения треков (дополнительно)

Время, используемое для разбиения треков. Если входные точки или полигоны имеют продолжительность, превышающую время разбиения, они будут разделены на несколько треков.

Если вы задали время и расстояние разбиения, треки будут разделены по одному или обоим условиям.

Предположим, у вас есть точечные объекты летящих самолетов, где поле трека соответствует ID самолета. Это самолет может выполнить несколько полетов, которые будут представлены как один трек. Если вы знаете, что между полетами имеется промежуток в 1 час, можно использовать 1 час для разбиения трека, тогда каждый полет будет находиться в собственном треке.

Разделение треков (дополнительно)

Вы можете разделять треки с помощью трех различных методов. Вы можете использовать методы разделения: нет, все или некоторые.

Разделение может быть выполнено одним из следующих образов:

  • На основе расстояния между входными данными – если входные точки или полигоны отдалены друг от друга на расстояние, которое превышает указанное расстояние разбиения, то они будут разделены на несколько треков. Например, если вы задали расстояние разбиения 10 километров, если в последовательности точек расстояние между точками превысит 10 километров, то в этом месте будет разрыв трека.
  • На основе времени между входными данными – если входные точки или полигоны записаны с перерывом времени, который превышает указанное время, то они будут разделены на несколько треков. Например, у вас есть точечные объекты летящих самолетов, где поле трека соответствует ID самолета. Это самолет может выполнить несколько полетов, которые будут представлены как один трек. Если вы знаете, что между полетами имеется промежуток в 1 час, можно использовать 1 час для разбиения трека, тогда каждый полет будет находиться в собственном треке.
  • В заданные интервалы времени – для разбиения используются регулярные интервалы, которые указываются интервалом времени и базовым временем. Если вы не укажите базовое время, то будет использоваться 1 января 1970. Например, вы указали 1 год с базовым временем 2 февраля 1990 года в 10:00, треки будут разделены на моментах времени 2 февраля 1990 года в 10:00, 2 февраля 1991 года в 10:00 и так далее с интервалом в 1 год.

Если вы задали несколько опция разбиения, треки будут разделены по одному или обоим условиям.

Выбрать расстояние для разбиения треков (дополнительно)

Расстояние, используемое для разбиения треков. Если входные точки или полигоны имеют продолжительность, превышающую время разбиения, они будут разделены на несколько треков.

Если вы задали время и расстояние разбиения, треки будут разделены по одному или обоим условиям.

Добавить статистику (дополнительно)

Вы можете вычислить статистику по суммируемым объектам.Вы можете вычислить следующее для числовых полей:

  • Количество – вычисляет количество ненулевых значений. Может использоваться с числовыми или текстовыми полями. Количество [null, 0, 2] – это 2.
  • Сумма – сумма числовых значений в поле. Сумма [null, null, 3] равна 3.
  • Среднее – среднее арифметическое числовых значений. Среднее [0, 2, null] равно 1.
  • Мин – минимальное значение числового поля. Минимальное значение [0, 2, null] равно 0.
  • Макс – максимальное значение числового поля. Максимальное значение [0, 2, null] равно 2.
  • Диапазон – диапазон числового поля. Он вычисляется вычитанием минимального значения из максимального. Диапазон [0, null, 1] равен 1. Диапазон [null, 4] равен 0.
  • Дисперсия – дисперсия по числовому полю в треке. Дисперсия [1] равна null. Дисперсия [null, 1,0,1, 1] равна 0,25.
  • Среднеквадратическое отклонение – среднеквадратическое отклонение числового поля. Среднеквадратическое отклонение [1] равно null. Среднеквадратическое отклонение [null, 1,0,1, 1] равно 0,5.

Вы можете вычислить следующее для числовых полей:

  • Количество – количество не пустых строк.
  • Любая – эта статистика является случайной выборкой строкового значения в указанном поле.
Вся статистика вычисляется на ненулевых значениях. Итоговый слой будет содержать новое поле для каждой вычисленной статистики. Можно добавить любое число вычисленных статистических значений, выбрав атрибут и статистику.

Выбрать ArcGIS Data Store, куда будут сохранены результаты

Результаты GeoAnalytics хранятся в хранилище данных и выглядят как векторный слой в Portal for ArcGIS. В большинстве случаев результаты следует сохранять в хранилище пространственно-временных данных, это настройка по умолчанию. В некоторых случаях , сохранение результатов в реляционном хранилище данных является хорошим решением. Следующее может быть поводом для сохранения результатов в реляционном хранилище данных:

  • Вы можете использовать результаты на портале-для-совместной работы.
  • Вы можете включить синхронизацию для результатов.

Вы не должны использовать реляционные хранилища данных, если ожидаете , что результаты GeoAnalytics увеличатся, и хотите использовать преимущества возможностей хранилища больших пространственно-временных данных для управления большими объемами данных.

Имя слоя результата

Имя создаваемого слоя. Если вы записываете в ArcGIS Data Store, ваши результаты будут сохранены в Моих ресурсах и добавлены на карту. Если вы записываете в файловое хранилище больших данных, ваши результаты будут сохранятся в файловом хранилище больших данных и добавляться в его файл манифеста. Они не будут добавлены на карту. Имя слоя по умолчанию зависит от имени инструмента и имени входного слоя. Если слой уже существует, произойдет сбой.

При записи в ArcGIS Data Store (реляционное или пространственно-временное хранилище больших данных) с помощью ниспадающего списка поля Сохранить результат в вы можете задать имя папки в разделе Мои ресурсы, в которую будет записан результат.
